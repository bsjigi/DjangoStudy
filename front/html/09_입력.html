# input tag

  • 사용자가 원하는 값을 입력할 수 있는 칸을 생성한다
  • type 속성에 따라 다양한 방식으로 입력을 받을 수 있다
  • input태그들은 폼(form)단위로 묶여있어야 한다
  • 반드시 name 속성을 통해 카테고리를 설정해야 한다

# form tag

  • input태그들을 묶는 태그
  • submit, reset 버튼은 폼(form)단위로 전송/취소를 수행한다
  • action 속성: submit 발생 시 데이터와 함께 이동할 페이지의 URL
  • method 속성: 전송 방식 설정 (GET/POST/PUT/...)
    GET 방식 - 데이터를 URL뒤에 붙여서 전송한다
    POST 방식 - 데이터를 request body에 실어서 전송한다
  • 데이터 전송시, input태그의 name속성값이 key역할을 하게 된다.
  • 데이터 전송시, input태그에 사용자가 입력한 값 또는 value속성이 value역할을 하게 된다.
타입 모양 설명
type="text" 기본적인 입력 칸
type="password" 비밀번호 입력 칸
type="number" 숫자만 입력할 수 있는 입력 칸
type="date" 날짜를 선택할 수 있는 입력 칸
type="radio"









여러개의 항목 중 하나만 선택할 수 있는 버튼을 생성한다.
같은 name 속성값을 지닌 버튼들 중에서 하나만 선택할 수 있다.
type="checkbox"




여러개의 항목을 선택할 수 있는 체크박스를 생성한다.
type="submit" 데이터를 전송하는 버튼을 생성.
버튼이 속한 폼 내부의 input태그들에 입력된 데이터들을 전송한다.
type="reset" 데이터 입력을 초기화하는 버튼을 생성.
버튼이 속한 폼 내부의 input태그들에 입력된 데이터들을 초기화한다.
<select> 사용자가 여러가지 중 하나를 선택할 수 있다
<textarea> 긴 글을 입력할 수 있는 영역을 생성한다
GET 방식에는 전송 길이 제한이 있기 때문에 (URL길이제한)
너무 긴 글은 다른 방식으로 보내는것이 좋다
