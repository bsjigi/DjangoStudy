# HTML의 기능은 중요하지 않다

  • HTML의 레이아웃 요소들은 이름만 그럴싸하고 아무런 기능이 없다
  • 하지만, 나중에 CSS를 통해 해당 요소들을 원하는대로 배치할 수 있다
  • HTML은 사이트의 뼈대 역할을 하기 때문에, 기능보다는 태그 이름에 담긴 뜻이 더 중요하다
  • HTML은 의미론적(semantic)이다
  • 나중에는 기능이 있는 태그들의 기능도 삭제해버리고 의미만 가져가는 경우도 많다
