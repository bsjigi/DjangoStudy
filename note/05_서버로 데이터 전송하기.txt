
# HTML 폼
    - 사용자가 input태그에 입력한 데이터를 폼 단위로 묶어서 서버로 전송한다
    - 서버로 전송할 때의 방식은 GET방식과 POST방식이 있다

# GET
    - GET방식은 URL뒤에 데이터가 함께 실려가는 방식
    - URL맨 끝에 ?name=value&.. 가 붙어서 데이터를 표현한다
    - 사용자가 마음대로 데이터를 보낼 수 있기 때문에 주의해야한다
    - 즐겨찾기 등록시에 사용했던 값들이 그대로 반영된다는 장점이 있다
    - 전송된 값들은 request.GET에 들어있다

# POST
    - POST방식은 요청의 내부에 데이터가 실려가는 방식이다 (주소에 값이 노출되지 않는다)
    - POST방식은 길이에 제한이 없어서 많은 양의 데이터를 전송할 수 있다 (파일 전송도 가능)
    - 즐겨찾기에 반영되지 않는다는 단점이 있다
    - Django 템플릿에서 POST방식을 통해 데이터를 전송할때는
      폼 내부에 {% csrf_token %} 템플릿 태그가 필요하다. (Djnago의 기본 보안 설정)
    - 전송된 값들은 request.POST에 들어있다
