
# 프로그램 개발 방식이 필요한 이유
    - 정해진 개발 방식 없이 프로젝트 진행시, 혼란이 발생한다.
    - 코드가 뒤섞여 유지보수가 힘들어지고, 생각지 못한 부분에서 에러가 많이 발생한다.
    - 규모가 커질수록 더욱 힘들어진다.

# MVC 패턴
    - 대표적인 프로그램 개발 방식
    - 프로그램을 구성하는 모든 코드들을 각자의 특징에 따라
      Model, View, Controller의 세 가지 형태로 분류하여 개발을 진행하는 방식

# Model
    - 데이터 클래스
    - 프로그램에서 자주 사용하게 되는 데이터의 형태를 정의해놓은 코드

# View
    - 사용자에게 보여지는 화면을 구성하기 위한 코드

# Controller
    - 사용자에게 보여지지 않는 내부적인 계산을 담당하는 코드

# Django의 MTV 패턴
    - models.py : MVC에서의 모델 역할
    - templates : MVC에서의 뷰 역할
    - views.py  : MVC에서의 컨트롤러 역할
